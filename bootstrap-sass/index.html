Pros

  • Improves search engine optimization because Google will soon move to a mobile first index
  • Better user experience
  • Saves time developing common components from scratch
  • Has great documentation and design system to follow
  • Smaller code footprint. More control to turn off unused componets and build a custom Bootstrap theme with the variables.
  • Quicker load time by merging and minifying code into one file and lowering the number of http requests
  • Quicker development time because it lessens the need for style overrides. You can change colors for all components with one line change.
Button

Cons

  • A learning curve. Need to follow a system and naming convention you didn't write yourself. Also need to learn Sass, command line, and the workflow to compile Sass into CSS.
  • Discipline to compile Sass for every style change and each team member needs to know and follow the workflow.
